# 示例网络使用说明

## 概述

`example_network.json` 是一个完整的杀伤链测试网络，包含所有必要的节点和连接。

## 网络结构

### 我方节点（蓝方）

1. **预警雷达-01** (传感器)
   - 探测范围：300km
   - 探测概率：90%
   - 位置：(150, 200)

2. **侦察卫星-01** (传感器)
   - 探测范围：500km
   - 探测概率：85%
   - 位置：(150, 300)

3. **指挥中心-01** (指挥节点)
   - 处理能力：500
   - 决策时延：3秒
   - 指挥范围：400km
   - 位置：(250, 200)

4. **导弹发射车-01** (打击单元)
   - 打击范围：250km
   - 毁伤率：85%
   - 精度：15m CEP
   - 位置：(350, 150)

5. **战斗机-01** (打击单元)
   - 打击范围：200km
   - 毁伤率：75%
   - 精度：20m CEP
   - 位置：(350, 250)

### 敌方节点（红方）

1. **敌方指挥部**
   - 位置：(550, 200)
   - 高价值目标

2. **敌方防空阵地**
   - 位置：(550, 300)
   - 威胁性目标

3. **敌方雷达站**
   - 位置：(550, 100)
   - 关键目标

## 杀伤链路径

该网络包含 **6 条完整杀伤链**：

### 对敌方指挥部的杀伤链
1. 预警雷达-01 → 指挥中心-01 → 导弹发射车-01 → 敌方指挥部
2. 侦察卫星-01 → 指挥中心-01 → 导弹发射车-01 → 敌方指挥部
3. 预警雷达-01 → 指挥中心-01 → 战斗机-01 → 敌方指挥部
4. 侦察卫星-01 → 指挥中心-01 → 战斗机-01 → 敌方指挥部

### 对敌方防空阵地的杀伤链
5. 预警雷达-01 → 指挥中心-01 → 导弹发射车-01 → 敌方防空阵地
6. 侦察卫星-01 → 指挥中心-01 → 导弹发射车-01 → 敌方防空阵地

### 对敌方雷达站的杀伤链
7. 预警雷达-01 → 指挥中心-01 → 导弹发射车-01 → 敌方雷达站
8. 侦察卫星-01 → 指挥中心-01 → 导弹发射车-01 → 敌方雷达站
9. 预警雷达-01 → 指挥中心-01 → 战斗机-01 → 敌方雷达站
10. 侦察卫星-01 → 指挥中心-01 → 战斗机-01 → 敌方雷达站

## 使用方法

### 在仿真推演中导入

1. 打开仿真推演页面
2. 点击"📁 导入网络文件"按钮
3. 选择 `example_network.json` 文件
4. 网络将自动加载，节点排列在画布上

### 测试杀伤链搜索

1. 导入网络后，**左键点击任意敌方节点**（红色）
2. 系统将自动搜索所有可用杀伤链
3. 选择最优杀伤链并自动执行打击
4. 观察打击效果和节点移动

### 测试节点移动系统

1. 点击"▶ 启动移动"按钮
2. 使用快捷战术指令：
   - 🔄 自动巡逻：打击单元开始矩形巡逻
   - 🛡 防御阵型：打击单元进入防御线
   - ⚔ 进攻阵型：打击单元向敌方中心扇形推进
3. 手动控制：
   - 左键拖拽我方节点移动
   - 右键点击画布设置移动目标

## 性能参数说明

### 传感器节点性能参数
- `detectionRange`: 探测范围（km）
- `detectionProbability`: 探测概率（0-1）
- `antiJamming`: 抗干扰能力（0-1）
- `reliability`: 可靠性（0-1）

### 指挥节点性能参数
- `processingCapacity`: 处理能力（任务/小时）
- `decisionDelay`: 决策时延（秒）
- `commandRange`: 指挥范围（km）
- `commDistance`: 通信距离（km）
- `reliability`: 可靠性（0-1）

### 打击节点性能参数
- `strikeRange`: 打击范围（km）
- `damageRate`: 毁伤率（0-1）
- `accuracy`: 精度（CEP米）
- `commDistance`: 通信距离（km）
- `reliability`: 可靠性（0-1）

## 预期效能

基于节点性能参数，该网络的预期效能：

- **最优杀伤链效能**：约 60-70%
- **协同效能**：约 90%+（多条杀伤链协同）
- **平均效能**：约 50-60%

## 调试提示

如果遇到问题：

1. **杀伤链搜索失败**
   - 检查节点是否有完整的 `performance` 属性
   - 检查边类型是否正确（detection/communication/strike）
   - 确保有完整的传感器→指挥→打击→目标路径

2. **效能显示 NaN%**
   - 现已修复：确保使用最新代码
   - 性能参数必须是数字类型，不能是字符串

3. **节点移动异常**
   - 确保启动了移动系统
   - 只有打击单元会响应战术指令
   - 手动拖拽在移动系统启用时才可用

## 创建自定义网络

参考此示例文件的结构，创建自己的网络时：

1. 确保每个节点都有 `performance` 对象
2. 根据节点 `baseType` 添加相应的性能参数
3. 创建完整的杀伤链路径：传感器 → 指挥 → 打击 → 目标
4. 边的 `type` 必须与连接类型匹配
